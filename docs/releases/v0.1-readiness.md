## 제품 정의

Flowness는 Issue-Driven AI Development Operating System이다. 요청을 Issue로 바꾸고, 각 Issue에 Log, Decision, Evidence, Review, Recovery, Human Gate를 연결해서 작업 과정을 추적 가능하게 만든다.

## v0.1 범위

### 포함 범위

- TypeScript-first npm workspace
- `flowness init`, `issue:create`, `request:create`, `workflow:create`, `workflow:validate`, `workflow:step`, `workflow:recover`
- `decision:create`, `review:run`
- `skill:create`, `skill:run`, `skill:list`
- `rule:create`, `rule:apply`, `rule:list`
- `config:gate`, `validate`, `upgrade`

- 이유: `AGENTS.md`와 `.agent/` 감사/릴리스 산출물이 아직 커밋 전이다.
- 최종 release cut 시점에는 working tree clean 상태로 정리해야 한다.

## Known debt

- 루트 `npm test`는 POSIX `find`에 의존한다.
- CLI orchestration이 `packages/cli/src/index.ts`에 많이 집중돼 있다.
- non-JS workflow loading은 runtime/subprocess 경로를 사용한다.

## v3 out-of-scope

- 제품 재설계 없음
- 새로운 roadmap line 없음
- release readiness 범위를 넘는 기능 추가 없음

## Audit decision

`READY WITH RISKS`

현재 증거로는 build/test/CLI/init/validate/audit 경로가 모두 확인되었고, release evidence와 audit matrix도 추가되었다. 다만 현재 작업 트리는 아직 커밋 전이므로, 최종 release cut 전에는 clean tree로 정리해야 한다.
