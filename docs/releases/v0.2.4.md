# Flowness v0.2.4 Release Notes

- Date: `2026-06-21`
- Scope: 패치 릴리스 (규칙 승인 흐름, 중앙 규칙 로그, 성능 트러블슈팅 문서, 업그레이드 보완)

## 요약 (Summary)

`flowness init` 이후에는 코딩 에이전트와 자연어로 작업하는 흐름을 전면에 두고, 명령은 탐색·검증·복구용 escape hatch로 정리했습니다. 규칙 변경은 자동 파일 수정 대신 승인 프롬프트를 거치며, 승인된 변경 이력은 `.flowness/rules/rule-update-log.md` 에만 기록됩니다. 또한 성능 작업을 위한 `docs/troubleshooting/performance-improvements.md` 를 추가하고, 업그레이드 시 누락 규칙/워크플로 파일이 검증 실패를 일으키지 않도록 보완했습니다.

## Added

- `docs/troubleshooting/performance-improvements.md`
- 규칙 변경 승인 프롬프트와 중앙 규칙 업데이트 로그 연동

## Changed

- `flowness init` 이후의 기본 작업 흐름을 자연어 중심으로 재배치했습니다.
- `.flowness/rules/*.md` 규칙 파일을 current-state 문서로 재생성하도록 정리했습니다.
- `rule:create` / `rule:update` 가 기존 규칙과 강하게 매칭되면 중복 파일 대신 기존 파일을 갱신하도록 바뀌었습니다.

## Fixed

- `flowness upgrade` 시 워크스페이스에 존재하지 않는 필수 `compareOnlyArtifacts`(규칙 및 워크플로 템플릿) 파일들을 누락 파일 목록(`addIfMissing`)에 추가하여 안전하게 업그레이드되도록 버그를 수정했습니다.
- 개별 규칙 파일 내부의 append-only 업데이트 블록을 제거하고, 이력은 `rule-update-log.md` 에만 append-only 로 남기도록 수정했습니다.

## Migration Notes

- 기존 프로젝트에서 `flowness upgrade --apply`를 다시 수행하면, 누락되었던 `.flowness/rules/git.md` 등의 파일이 안전하게 추가되며 이후 `flowness validate` 통과가 가능해집니다.
- 새 규칙 승인 흐름을 쓰려면 `flowness request:create` 로 규칙 변경 후보를 확인한 뒤 승인에 따라 `flowness rule:update` 를 사용하세요.
